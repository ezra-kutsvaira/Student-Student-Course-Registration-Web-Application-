# Contributing to Documentation

## Overview
This document explains how contributors should add or update project documentation.

## When to Update Documentation
Update the docs when changing:

- features,
- setup instructions,
- environment variables,
- database requirements,
- user workflows,
- Docker behavior,
- project structure.

## Contribution Rules
- Edit the most relevant document instead of duplicating content.
- Use consistent terminology.
- Keep instructions clear and step-based.
- Verify commands before documenting them.
- Link related documents when useful.

## Review Checklist
Before submitting documentation changes:

- check spelling and grammar,
- confirm commands are correct,
- confirm the content matches current behavior,
- remove outdated information,
- update screenshots if needed.
